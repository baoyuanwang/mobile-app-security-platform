## 1. 本章目标

本章介绍移动应用安全检测平台（Mobile Application Security Analysis Platform，MASAP）的建设背景、行业现状、问题定义、设计目标、适用范围以及文档组织结构。

阅读本章后，读者应能够理解：

- 为什么需要建设统一的移动应用安全检测平台；
- 当前移动应用安全检测技术存在的主要问题；
- MASAP 的总体定位及设计原则；
- 本白皮书的研究范围与目标读者；
- 整个平台后续章节之间的逻辑关系。

# 2. 建设背景

近年来，移动互联网已成为数字社会最重要的基础设施之一。Android 与 HarmonyOS 已形成覆盖全球数十亿终端设备的移动生态，应用市场每天接收大量应用及版本更新，涵盖金融、政务、医疗、教育、电商、社交、游戏等多个行业。

随着应用规模不断扩大，移动应用的安全风险也持续演进，风险来源逐渐由传统病毒扩展到更加复杂的供应链攻击、恶意 SDK、动态加载、云控木马、隐私违规、诈骗应用、恶意广告、仿冒应用以及 AI 自动生成恶意代码等新型威胁。

应用商店、安全厂商以及终端厂商面临以下共同挑战：

- 应用数量持续增长，人工审核成本快速上升；
- 恶意行为越来越倾向于运行时触发，传统静态检测难以发现；
- 加固、混淆、动态加载等技术大量使用，增加程序分析难度；
- 第三方 SDK 已成为隐私违规、供应链攻击的重要入口；
- 大模型技术推动恶意代码自动生成，传统特征检测逐渐失效。

因此，移动应用安全检测已经从传统的病毒扫描，逐步发展为融合程序分析、动态行为分析、人工智能、安全知识管理及自动化审核于一体的综合分析平台。

---

# 3. 行业现状

目前业内主流移动应用安全检测平台通常包括以下几类：

| 平台类型 | 代表能力 | 局限性 |
|-----------|----------|----------|
| 应用商店审核平台 | 应用上架审核 | 检测能力偏业务化 |
| 安全厂商检测平台 | 恶意软件分析 | AI 与程序分析结合不足 |
| 终端安全平台 | 终端防护 | 偏运行时防御 |
| 企业移动安全平台 | 企业应用管理 | 缺乏大规模检测能力 |

虽然不同平台定位不同，但其核心能力通常围绕以下几个方面展开：

- 应用安装包解析；
- 静态程序分析；
- 动态行为分析；
- AI 图像与文本识别；
- 风险规则匹配；
- 人工审核辅助。

然而，这些能力大多由多个独立系统组成，缺乏统一的架构设计和数据模型，导致分析结果难以共享，能力复用程度较低。

# 4. 问题定义（Problem Statement）

当前移动应用安全检测平台主要面临以下技术问题。

## 4.1 检测能力碎片化

静态分析、动态分析、AI 检测、人工审核通常分别部署在不同系统中。

由于缺乏统一的数据模型，不同系统之间难以共享分析结果，导致：

- 重复计算；
- 检测链路冗长；
- 风险关联困难；
- 人工复核成本高。

---

## 4.2 程序分析能力不足

大量检测平台仍停留在 APK 解包、权限扫描、字符串匹配等传统方式。

对于以下复杂场景支持有限：

- 动态加载（Dynamic Loading）
- Native 代码分析
- Reflection
- 多 Dex
- Harmony HAP
- 动态特性模块
- 代码混淆
- 加固保护

程序语义理解能力成为平台能力的重要瓶颈。

---

## 4.3 AI 与检测流程割裂

近年来，OCR、CV 以及大语言模型逐渐应用于安全检测。

但多数平台中 AI 仅承担页面识别、文本分类等辅助任务。

AI 无法直接理解程序分析结果，例如：

- Call Graph
- Data Flow
- Control Flow Graph（CFG）
- 生命周期模型
- Runtime Behavior

导致 AI 无法真正参与安全推理。

---

## 4.4 知识无法持续沉淀

检测平台每天都会产生大量安全数据，包括：

- IOC（Indicator of Compromise）
- 恶意样本
- SDK 行为
- 域名
- IP
- 开发者信息
- 数字证书
- 检测证据

然而，大部分平台仅保存检测报告，没有形成统一的知识平台，无法支持持续学习和长期演进。

---

## 4.5 自动化能力不足

移动应用安全检测越来越依赖自动化执行。

传统 Monkey 测试已无法覆盖复杂业务逻辑，例如：

- 登录流程；
- 权限弹窗；
- 金融交易；
- 多页面跳转；
- 云控触发。

未来需要结合状态机、OCR、LLM Agent 等技术实现智能自动化分析。

---

# 5. MASAP 平台定位

MASAP（Mobile Application Security Analysis Platform）定位于企业级移动应用安全检测平台。

平台覆盖应用从提交、审核、发布到运营的全生命周期安全检测过程。

平台不仅关注恶意软件检测，同时覆盖：

- 隐私合规；
- 恶意广告；
- 内容安全；
- 仿冒侵权；
- 涉诈风险；
- SDK 风险；
- 软件供应链风险。

MASAP 的核心定位包括：

- 统一程序分析平台；
- 统一检测平台；
- AI 原生分析平台；
- 安全知识平台；
- 企业级审核平台。

---

# 6. 设计目标

MASAP 的总体建设目标如下。

## Goal 1：统一分析引擎

建立统一 Program Analysis Engine。

所有检测能力均建立在统一分析结果之上。

---

## Goal 2：AI Native

AI 不是外挂，而是平台核心组成部分。

AI 参与：

- 页面理解；
- 行为解释；
- 风险推理；
- 报告生成；
- 规则学习。

---

## Goal 3：Knowledge Driven

建立统一 Security Knowledge Platform。

所有检测结果均沉淀为长期知识资产。

---

## Goal 4：Detection as a Service

所有检测能力均服务化。

通过统一 API 对外开放。

---

## Goal 5：Cloud Native

平台支持：

- Kubernetes；
- 容器部署；
- 微服务；
- 弹性扩缩容；
- 分布式调度。
